# 🔥 Project Phoenix

> *A fully automated, AI-powered Windows workstation that can rebuild itself from a clean Windows installation.*

---

## Vision

Project Phoenix is more than a collection of PowerShell scripts.

It is an Infrastructure-as-Code solution for Windows workstations, designed to automate the installation, configuration, maintenance, and ongoing management of a complete development, AI, and gaming environment.

The goal is simple:

> Describe the ideal workstation once, then allow Project Phoenix to build, maintain, repair, and evolve it automatically.

Rather than documenting hundreds of manual configuration steps after every Windows reinstall, Project Phoenix transforms those preferences into code.

---

## Core Principles

Project Phoenix is built around several key principles.

### Automation First

Every repetitive task should be automated.

If a process must be repeated more than once, it should become part of Project Phoenix.

---

### Infrastructure as Code

The workstation itself is treated as software.

Applications, settings, folder structures, developer tools, AI environments, and gaming libraries are all defined through version-controlled configuration.

---

### Modular Architecture

Each feature is implemented as an independent module.

Examples include:

* Windows Configuration
* Application Installation
* Development Environment
* AI Tooling
* Gaming Libraries
* Health Monitoring
* Reporting
* Backup & Recovery

Each module can evolve independently while remaining part of the same ecosystem.

---

### Idempotent Design

Running Project Phoenix multiple times should never damage the workstation.

Instead, every execution compares the desired state with the current state, making only the changes required to bring the system back into compliance.

---

### Human Friendly

Automation should remain understandable.

Every module should be documented, readable, and easy to maintain.

Project Phoenix is intended to help users learn how their workstation operates rather than hiding everything behind complex automation.

---

## Project Goals

* Rebuild a complete Windows workstation from a clean installation.
* Configure development tools automatically.
* Configure AI tooling automatically.
* Standardise gaming libraries across launchers.
* Detect duplicate software and unnecessary storage usage.
* Monitor workstation health.
* Maintain consistent configuration over time.
* Reduce manual setup from hours to minutes.

## Long-Term Vision

The ultimate goal is for a freshly installed Windows computer to become a fully configured workstation by running a single command.

Project Phoenix will:

* Install required applications.
* Configure Windows.
* Configure development environments.
* Configure AI tooling.
* Configure gaming launchers.
* Restore personal preferences.
* Validate system health.
* Produce a complete installation report.

The result is a workstation that is predictable, reproducible, maintainable, and ready for work within minutes.
