# Guides

## Purpose

Task-oriented, how-to documentation for people using or extending Project Phoenix — distinct from the architectural "why" material in [docs/adr](../adr/README.md) and [docs/architecture](../architecture/README.md).

## Scope

Practical walkthroughs: how to add a new module, how to write a Phoenix-compliant configuration file, how to run the test suite locally, how to add a new application to the installer config. Not for design rationale or troubleshooting specific failures — see [docs/troubleshooting](../troubleshooting/README.md) for the latter.

## Expected Contents

Short, focused guides, one task per file, added as real workflows emerge from Sprint 1 onward. Currently empty.
