# Troubleshooting

## Purpose

Captures known failure modes and how to diagnose them, so the same problem doesn't get re-investigated from scratch every time it resurfaces.

## Scope

Symptom-to-cause documentation: what an error message means, why a module reports an unhealthy status, why CI failed in a particular way. Distinct from [docs/recovery](../recovery/README.md), which covers what to do once a problem is understood, and from [docs/guides](../guides/README.md), which covers normal usage rather than failures.

## Expected Contents

One entry per recurring or non-obvious failure mode, added as they're encountered and diagnosed during development and real-world use. Currently empty — no automation exists yet to fail in interesting ways.
