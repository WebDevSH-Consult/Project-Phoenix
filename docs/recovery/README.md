# Recovery

## Purpose

Documents how to recover a workstation, a module's state, or a configuration after something has gone wrong — the operational counterpart to the rest of the documentation, which focuses on building forward.

## Scope

Rollback procedures, manual recovery steps for when automation fails partway through, and guidance for restoring from backups once the Backup Manager module exists. Does not cover diagnosing why something failed — see [docs/troubleshooting](../troubleshooting/README.md) for that.

## Expected Contents

Recovery runbooks, added as modules that can leave the system in a partially-modified state are built (Windows Configuration, Application Installer, Backup Manager). Currently empty — there is no automation yet that needs a recovery path.
